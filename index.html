English version below ↓

Esta Política de Privacidade descreve de forma completa como a extensão Conceptum IA ("extensão") coleta, usa, processa, armazena e compartilha informações. Ao instalar ou utilizar a extensão, você concorda com as práticas descritas neste documento.

Resumo: A Conceptum IA não coleta dados pessoais identificáveis além dos estritamente necessários para validar a licença de uso. O conteúdo dos processos judiciais é enviado diretamente ao provedor de IA escolhido pelo usuário e nunca transita ou é armazenado em servidores próprios da extensão.

3. Como os dados são processados

3.1 Validação de licença

Quando o usuário insere a chave de licença, a extensão calcula o fingerprint do dispositivo localmente (no navegador) e envia o par chave de licença + fingerprint ao servidor de autenticação via HTTPS. O servidor verifica se a chave é válida e se o dispositivo está autorizado, retornando apenas uma resposta de aprovação ou rejeição. Nenhum outro dado é transmitido nesse processo.

3.2 Análise de processos

O texto extraído de PDFs é transmitido diretamente da extensão para a API do provedor de IA selecionado pelo usuário, usando a chave de API do próprio usuário. A extensão atua como intermediária técnica — o conteúdo nunca passa por servidores próprios da extensão e não é armazenado pela extensão em nenhum momento.

3.3 Armazenamento local

A extensão utiliza chrome.storage.local para armazenar, exclusivamente no dispositivo do usuário:

  • Chave de licença inserida pelo usuário;
  • Chaves de API dos provedores de IA;
  • Preferências e configurações da extensão (ex.: provedor de IA padrão, modelo selecionado);
  • Token de sessão do Supabase (se o usuário criar conta).

Esses dados permanecem no dispositivo até que o usuário desinstale a extensão ou limpe o armazenamento nas configurações do navegador. Eles não são sincronizados entre dispositivos nem transmitidos a terceiros, exceto nas situações descritas nesta política.

7. Segurança

  • Todas as comunicações com serviços externos utilizam HTTPS/TLS.
  • O fingerprint do dispositivo é calculado localmente como hash SHA-256 e transmitido apenas na forma de hash — os dados originais (CPU, memória, etc.) nunca deixam o dispositivo.
  • As chaves de API dos provedores de IA são armazenadas somente no armazenamento local do navegador, protegido pelo perfil do usuário no sistema operacional, e nunca são transmitidas a servidores da extensão.
  • Em caso de incidente de segurança que afete dados pessoais, os usuários afetados serão notificados por e-mail (se disponível) ou por meio de atualização publicada nesta página, no prazo cabível.

8. Cookies e rastreamento

A extensão não utiliza cookies, pixels de rastreamento, impressões digitais de navegador para fins publicitários, ferramentas de análise comportamental (como Google Analytics) nem qualquer outra tecnologia de rastreamento entre sites. O fingerprint de dispositivo descrito nesta política é usado exclusivamente para validação de licença e nunca para rastreamento de comportamento do usuário.

9. Dados de menores

Esta extensão é destinada exclusivamente a profissionais do direito e não coleta intencionalmente dados de menores de 18 anos. Se identificarmos que dados de menor foram inadvertidamente coletados, os excluiremos imediatamente.

Na qualidade de titular dos dados, você tem os seguintes direitos garantidos pela Lei Geral de Proteção de Dados:

  • Exclusão: solicitar a eliminação dos dados tratados com base em seu consentimento.

  • Revogação do consentimento: revogar o consentimento a qualquer momento, sem prejuízo das operações já realizadas.

Para exercer qualquer desses direitos, entre em contato pelo e-mail rrc_temp-conceptum@yahoo.com. Dados armazenados localmente podem ser excluídos diretamente pelo usuário desinstalando a extensão ou limpando o armazenamento nas configurações do Chrome (chrome://settings/content/all).

3. How Data Is Processed

3.1 License validation

When the user enters the license key, the extension calculates the device fingerprint locally (in the browser) and sends the license key + fingerprint pair to the authentication server via HTTPS. The server verifies whether the key is valid and whether the device is authorized, returning only an approval or rejection response. No other data is transmitted in this process.

3.2 Case analysis

Text extracted from PDFs is transmitted directly from the extension to the AI provider's API selected by the user, using the user's own API key. The extension acts as a technical intermediary — the content never passes through the extension's own servers and is not stored by the extension at any point.

3.3 Local storage

The extension uses chrome.storage.local to store, exclusively on the user's device:

  • License key entered by the user;
  • AI provider API keys;
  • Extension preferences and settings (e.g., default AI provider, selected model);
  • Supabase session token (if the user creates an account).

This data remains on the device until the user uninstalls the extension or clears storage in the browser settings. It is not synced across devices or transmitted to third parties, except in situations described in this policy.
